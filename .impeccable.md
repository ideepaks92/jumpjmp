# Design Context — JumpJMP

## Target Audience
Manufacturing engineers, quality engineers, R&D engineers, and process engineers. People who currently use JMP, Minitab, or Excel for statistical analysis. They value precision and correctness but are frustrated by clunky desktop software. They work in offices, labs, and factory floors.

## Use Cases
- Upload a CSV/XLSX, immediately see a sortable data table
- Build interactive charts by dragging columns to axes
- Run SPC control charts and check process capability (Cpk)
- Fit distributions to fatigue/reliability data
- Share a live analysis link with a colleague — no install needed

## Brand Personality / Tone
**Precision-modern.** Think: a well-calibrated instrument with a clean digital display. Authoritative without being cold. Technical confidence with approachable warmth. Not corporate stiff, not startup playful. The feeling of a beautifully machined tool — functional, exact, satisfying to use.

## Quality Bar
MVP heading toward flagship. The demo page is the primary showcase — it must feel polished and intentional. The landing page is the first impression.

## Design Principles
- Data density matters — engineers work with numbers, don't waste space
- Precision over decoration — every pixel earns its place
- Trust through clarity — correct labels, proper sig figs, clear hierarchy
- Warmth through craft — tinted neutrals, considered typography, smooth transitions
